sympyprt (howto)

  • matplotlib (pylab) is not really required but is needed if the mplib mode is chosen or if running the testfiles. Use ipython qtconsole --pylab=inline if you want to use it.
  • Try latex --version and dvipng --version in a shell to assure that LATEX is working properly.
  • The following settings in ipython_qtconsole_config.py are recommended:
    1. c.IPKernelApp.pylab = True
    2. c.IPythonWidget.gui_completion = True
    3. c.IPythonWidget.buffer_size = 5000
  • Assure that the current directory (use pwd to show) has write access. Otherwise use cd to change to another directory.

Explanations:

  • ObjCache stores the TeX objects of the sympy instance by the id. In the example above, s is a sympy object while sT is a TeX2 object that stores the tex source the rendered png image (s knows nothing about sT). Why does sT show an image? Because it has a _repr_png_ method, while s is showing an image because of a special IPython printing method that recognizes sympy object automatically.

  • The TeX0, TeX1, TeX2 classes perform the production of the images. This class can be used on its own, for example:
